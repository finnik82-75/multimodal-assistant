# Мультимодальный Telegram-ассистент для отдела продаж ЗМГ

**Описание.** ИИ-консультант компании «Забайкальская медиа группа» в виде Telegram-бота. Консультирует рекламодателей по размещению рекламы (радио, портал ZAB.RU, ЗабТВ, соцсети), выявляет потребности и подводит к записи на консультацию с менеджером. Реализованы текстовый режим, RAG по базе знаний, голосовой ввод/ответ и анализ изображений.

**Задача.** Автоматизировать первичный контакт с клиентами: квалификация запроса, консультация по форматам и площадкам, направление в менеджерский воронку без раскрытия прайса в боте.

**Бизнес-результат.** Снижение нагрузки на менеджеров на типовых запросах, единая точка входа для рекламодателей в регионе, возможность масштабировать консультации за счёт базы знаний (RAG).

---

## Моя роль

Проектирование и реализация бота с нуля: архитектура пайплайна (handlers → router → OpenAI/ChromaDB), интеграция OpenAI API (LLM, Vision, STT, TTS), настройка RAG на ChromaDB и LangChain, системный промпт под сценарий отдела продаж ЗМГ, мультимодальные режимы и команды (/mode, /stats, /reset и др.).

---

## Архитектура

Пайплайн: **пользователь → handlers → services/router → OpenAI (LLM / Vision / STT / TTS) + ChromaDB (RAG) → ответ в Telegram.**

- **handlers** — приём сообщений (текст, голос, фото, документы) и команд, вызов router.
- **router** — выбор режима (text / rag / voice / vision), сборка контекста, вызов LLM или RAG-поиска.
- **rag** — загрузка и чанкинг документов (PDF, TXT, MD), индексация в ChromaDB, поиск по эмбеддингам, ответ с указанием источников.

---

## Режимы работы

| Режим | Описание |
|-------|----------|
| **text** | Диалог-консультант: выявление целей, формата и географии, подведение к записи на консультацию. Без RAG. |
| **rag** | Ответы только по документам из базы знаний, с указанием источника. При отсутствии данных — предложение связаться с менеджером. |
| **voice** | Голосовой ввод (STT), ответ текстом и опционально голосом (TTS). |
| **vision** | Анализ отправленных изображений в контексте запроса (OpenAI Vision). |

Переключение: `/mode text | rag | voice | vision`. Документы для RAG — в `data/documents/` (индексация при старте) или загрузка файлом в бота.

## Стек технологий

Python, pyTelegramBotAPI (AsyncTeleBot), OpenAI API (GPT-4o, Whisper, TTS, Vision), LangChain, ChromaDB, OpenAI Embeddings, PyPDF / LangChain TextLoader, python-dotenv.

---

## Как запустить

```bash
python -m venv venv
venv\Scripts\activate   # Windows
pip install -r requirements.txt
```

В корне создать `.env` с переменными `TELEGRAM_BOT_TOKEN` и `OPENAI_API_KEY`. Опционально: `USE_PROXYAPI`, `BOT_MODE`, `DEFAULT_VOICE`, `LOG_LEVEL`.

```bash
python main.py
```

Документы для RAG класть в `data/documents/` (PDF, TXT, MD). Статус базы — команда `/stats`.
